Functional Requirements:

  • Customer registration with Code, Name, and CNPJ
  • Customer address registration with geographic location
  • Linking the Address to the Customer
  • Visualization of customers on a map with filter options

Non-Functional Requirements:

  • The system must be secure, ensuring the integrity and confidentiality of information
  • The system must be responsive and user-friendly
  • The system must have a fast response time
  • The system must be scalable to handle a large number of customers and addresses

Business Rules:

  • A customer cannot be registered without a Name, CNPJ, and Address with a location
  • The Code, Name, and CNPJ fields must be mandatory in the customer registration
  • The customer's address must contain a geographic location
  • The address must be linked to the customer in the registration
  • The customer visualization map must allow filtering customers by name, CNPJ, and geographic location

Additional Considerations:

  • The system should allow storing multiple addresses for a single customer if necessary
  • The system should allow updating customer and address data, as well as their deletion
